| | NAS 80k | attempt7 (fine-tune) |
|---|---|---|
| 원본 | NAS 서버 학습 | 영어 base-960h → 한국어 fine-tune |
| Vocab | 1910 (음절 BPE) | 56 (자모) |

| Margin min | 0.050 | 0.025 |
| NB 크기 | 77MB | 72MB |
| 추론시간 | 424ms | ~400ms |
| uint8 결과 | 1~4음절만 출력 | 부분 한국어 출력 |

NAS 80k이 FP32에서 훨씬 좋지만 (WER 20% vs 39%), **uint8 T527 결과는 둘 다 실패** — margin min이 uint8 step보다 작기 때문.

## 실패 원인

```
FP32 logit margin min = 0.050
uint8 step size       = 0.160

margin min / step = 0.31x < 1.0

→ 가장 margin이 작은 non-blank 토큰들이 blank로 뒤집힘
→ 149 프레임 중 145~149가 blank
→ 음성 정보 대부분 소실
```

영어 모델은 margin min / step = 4.2x → 모든 non-blank 토큰 생존.

## 결론

1. **NAS 80k 모델은 FP32에서 매우 우수** (WER 20.14%, CER 9.48~35%)
2. **T527 uint8에서는 실패** — logit margin min 0.05 < uint8 step 0.16
3. **Transformer 기반 한국어 모델의 근본 한계** — 1910개 음절 vocab에서 logit을 충분히 벌려주지 못함
4. **QAT가 필요하지만**, 기존 QAT 실험 (margin mean 38% 개선)에서도 margin min은 개선 안 됨
5. **KoCitrinet (CNN 기반, CER 44.44%)이 여전히 T527 유일한 실용 한국어 STT**
