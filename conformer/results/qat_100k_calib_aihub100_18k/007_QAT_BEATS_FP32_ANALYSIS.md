# 007.저음질: QAT uint8이 FP32를 능가하는 현상 분석

**결론: 서버 FP32 CER 13.19% > T527 QAT uint8 CER 11.13% (QAT가 2.06%p 우세)**

11개 데이터셋 중 007.저음질만 QAT가 FP32를 역전. 나머지 10개는 FP32가 우세 (정상).

## 6. 핵심 발견: FP32의 짧은 저음질 발화 Hallucination

### ≤3s 발화에서 hyp 길이 분석

| 항목 | FP32 | QAT |
|------|------|-----|
| 평균 ref 길이 | 8.8자 | 8.8자 |
| **평균 hyp 길이** | **20.2자** | **8.7자** |
| hyp/ref 비율 | **2.30배** | **0.99배** |
| hyp < ref (deletion 우세) | 223개 (22.0%) | 154개 (15.2%) |
| **hyp > ref (insertion 우세)** | **745개 (73.6%)** | **54개 (5.3%)** |

**FP32 서버는 짧은 저음질 발화에서 ref의 2.3배 길이의 텍스트를 출력.**
평균 11.4자의 hallucination (없는 말 생성).
QAT는 hyp 길이가 ref와 거의 동일 — hallucination 없음.

| 네맞아요 (4자) | 해와봐야 (4자) | 100% | 네 맞아요 | 0% |

FP32가 입력 오디오와 전혀 무관한 텍스트를 출력하는 catastrophic failure 패턴.

## 9. 원인 분석

### 가설 1: Full-length 추론의 짧은 저음질 취약점 (주 원인, ~1.5%p 기여)

서버는 오디오 전체를 한 번에 Conformer에 입력. 3초 미만의 짧은 저음질 오디오에서:

1. **유효 음성 프레임 < 노이즈 프레임**: 전화 녹음의 배경 노이즈가 프레임 대부분을 차지
2. **CTC blank 부족**: FP32 모델이 노이즈 프레임에도 token을 배치 → insertion 폭발
3. **결과**: ref 8.8자인데 hyp 20.2자 출력 (2.3배 hallucination)

T527은 3초 윈도우로 고정 입력하므로:
- 짧은 오디오도 301프레임으로 패딩 → CTC가 빈 구간을 blank로 정상 처리
- **슬라이딩 윈도우가 우연히 짧은 저음질에 유리한 입력 포맷을 제공**

근거:
- ≤3s에서만 9.20%p 역전, >3s에서는 FP32 우세
- FP32 insertion 73.6% vs QAT 5.3%
- FP32 hyp 길이 20.2자 vs QAT 8.7자

### 가설 2: 양자화의 Regularization 효과 (~0.5%p 기여)

uint8 양자화는 weight에 균일 양자화 노이즈를 추가하는 효과:

1. **작은 activation 차이 억제**: FP32에서 노이즈가 만드는 미세한 activation peak가 uint8에서는 양자화 step 이하로 뭉개짐
2. **QAT FakeQuantize 학습**: 양자화 노이즈 존재 하에서 학습 → 노이즈에 robust한 weight 분포
3. **Dropout 유사 효과**: 양자화가 불필요한 미세 패턴을 제거

근거:
- catastrophic failure 제거 후에도 1.64%p 차이 유지
- 짧은 발화(ref<10자)에서 QAT exact match 62.2% vs FP32 40.6%
- 다른 데이터셋에서는 이 효과가 양자화 정밀도 손실보다 작아서 역전 안 됨

### 가설 3: mel 전처리 차이 (부차적, 정량화 어려움)

| 항목 | 서버 (NeMo) | T527 (C/KissFFT) |
|------|------------|------------------|
| FFT | torch.stft (FP32) | KissFFT (FP32) |
| Window | Hann (torch) | Hann (수동) |
| Mel filterbank | Slaney (librosa) | Slaney (수동 구현) |
| Normalize | per_feature | per_feature |

미세한 수치 차이가 저음질 환경에서 amplify될 가능성 있으나, 이것만으로 2%p 차이를 설명하기 어려움.

## 11. 결론

### 현상 정리

007.저음질에서 QAT uint8이 FP32보다 2.06%p 좋은 현상은 **3가지 요인의 복합 결과**:

1. **주 원인 (~1.5%p)**: 서버 FP32 full-length 추론이 짧은 저음질 발화에서 hallucination. 슬라이딩 윈도우 방식이 우연히 이 문제를 회피.
2. **부 원인 (~0.5%p)**: uint8 양자화의 regularization 효과가 저음질 노이즈 환경에서 spurious token 억제.
3. **배경 요인**: mel 전처리 차이가 저음질에서 미세하게 기여 가능.

### 논문 관점

- **"양자화가 FP32를 능가한다"는 일반화 불가** — 11개 중 1개 데이터셋, 특정 조건(저음질+짧은 발화)에서만 발생
- **의미있는 발견**: full-length CTC 추론이 짧은 저음질 입력에서 hallucination 취약점을 가짐. 슬라이딩 윈도우 + 양자화 regularization이 이를 완화
- **실용적 시사점**: 저음질 환경에서는 full-length보다 fixed-window 추론이 유리할 수 있음
- **재현 가능**: 동일 모델, 동일 데이터, 동일 CER 방식으로 수치 검증 완료

### 추가 검증 필요 사항

1. 서버에서 동일 3초 슬라이딩 윈도우 FP32 추론 → hallucination 사라지는지 확인
2. T527에서 full-length QAT 추론 (가능하면) → hallucination 발생하는지 확인
3. 007.저음질의 SNR 분포 측정 → SNR과 CER 차이의 상관관계

이 검증이 완료되면 가설 1 (full-length vs sliding window)을 확정할 수 있음.
