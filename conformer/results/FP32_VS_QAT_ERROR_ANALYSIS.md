## 2. 에러 유형 비율 (전체 합산)

| | FP32 | QAT | 변화 |
|---|------|-----|------|
| **Insertion** | 17.1% (5,980) | **24.4% (11,929)** | **+99.5% (2배)** |
| **Deletion** | 26.2% (9,174) | 28.7% (14,006) | +52.7% |
| **Substitution** | **56.7% (19,814)** | 46.9% (22,878) | +15.5% |

### 핵심 차이

- **FP32는 substitution 위주** (56.7%) — 비슷하지만 틀린 글자로 대체
- **QAT는 insertion이 2배 증가** (17.1% → 24.4%) — 없는 글자를 추가
- QAT의 I/D ratio가 전반적으로 높음 (0.3~0.6x → 0.7~1.0x)

## 3. QAT Insertion 증가 원인 1: 간투어 삽입

QAT가 발화 끝에 불필요한 토큰을 추가하는 패턴 발견.

| 추가 토큰 | 횟수 |
|----------|------|
| **"음"** | **83회** |
| "그" | 39회 |
| "아" | 37회 |
| "네" | 18회 |
| "요" | 15회 |
| "어" | 5회 |
| 기타 | 63회 |
| **합계** | **260회 (전체의 1.4%)** |

### 예시

| ref | QAT hyp | 추가 토큰 |
|-----|---------|----------|
| 감사합니다 좋은 하루 되세요 | 감사합니다 좋은 하루 되세요 **음** | "음" |
| 네 알겠습니다 | 네 알겠습니다 **그** | "그" |
| 확인해 드리겠습니다 | 확인해 드리겠습니다 **아** | "아" |

### 후처리로 제거 가능

```
규칙: 발화 끝 단독 음절 "음", "그", "아", "네", "요", "어" 제거
예상 효과: CER 0.3~0.5%p 개선
```

## 4. QAT Insertion 증가 원인 2: 슬라이딩 윈도우 경계 반복

멀티청크 처리 시 윈도우 경계에서 단어가 반복되는 패턴.

대부분은 원본 발화 자체의 반복 ("바로바로", "네네")이지만, 슬라이딩 윈도우 경계에서 발생하는 순수 반복도 존재.

## 7. 종합 분석

### FP32 서버의 장점
- **Substitution 비율 낮음** (56.7%) — 맞는 글자를 더 잘 선택
- **Exact match 30% 더 많음** — 정확히 맞추는 샘플이 압도적
- **멀티청크 경계 문제 없음** — full-length 추론이라 끝잘림/반복 없음

### FP32 서버의 약점
- **짧은 저음질 발화에서 hallucination** — 007.저음질 ≤3s에서 hyp 길이 2.3배
- **해당 구간에서 catastrophic failure** (CER≥80%) 30개

### QAT T527의 장점
- **짧은 저음질에서 robust** — hallucination 없음
- **고정 길이 입력으로 안정적** — 입력 길이에 무관한 성능

### QAT T527의 약점
- **Insertion 2배** — 간투어 "음" 83회, 경계 반복
- **Exact match 30% 적음** — 정밀도에서 FP32에 뒤짐
- **멀티청크 끝잘림** — 31개 발화에서 끝부분 소실

## 8. 개선 방안

| 문제 | 방안 | 기대 효과 |
|------|------|----------|
| 간투어 삽입 ("음" 83회) | CTC 디코드 후 끝 단독 음절 제거 | CER -0.3~0.5%p |
| 멀티청크 끝잘림 (31개) | 마지막 청크 유효 프레임 확인 + 패딩 | CER -0.1%p |
| Insertion 전반 증가 | Beam search + LM으로 spurious token 억제 | CER -0.5~1.0%p |
| Substitution (최대 에러 소스) | QAT loss 함수 개선, calib 분포 최적화 | CER -0.3~0.5%p |
