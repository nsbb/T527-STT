## 1. 검증 목적

18,368개 대규모 테스트는 vpm_run (CLI)으로 수행. 실제 서비스는 Android 앱.
**두 방식의 NPU 추론 결과가 동일한지** 검증하여 vpm_run 기반 실험 결과의 신뢰성 확보.

---

## 2. 검증 방법

1. 동일한 NB 파일을 디바이스에 push (md5 검증)
2. 동일한 mel chunk .dat 파일을 입력으로 사용
3. vpm_run → Python에서 output dequantize → argmax → CTC decode
4. 앱 → JNI에서 NPU 추론 → argmax → Java CTC decode
5. 문자열 결과 1:1 비교

### 테스트 데이터
- 7F_KSK + 7F_HJY 자체 수집 월패드 음성 10개
- CER 0% (정확히 맞는 것) ~ CER 7% (약간 틀리는 것) 분포

## 5. CSV vs vpm_run 불일치 2건 분석

### #2 (7F_KSK idx=3): "뭐지" vs "뭐지 번호"

| | 결과 |
|---|---|
| CSV (18k 테스트 당시) | 관리사무소 번호가 뭐지 |
| vpm_run (지금) | 관리사무소 번호가 뭐지 **번호** |

- mel 길이: 155 프레임 (1.5초)
- 실제 출력 프레임: 39개 (155÷4)
- 패딩 영역 (frame 39~75)의 frame 52, 54에서 "번호" 토큰 출력
- **원인:** 18k 테스트 당시와 지금의 NB 파일이 동일하나, 18k 테스트 스크립트가 실행 중 디바이스에 다른 NB가 로드되어 있었을 가능성
- **영향:** CER 변화 없음 (ref "관리사무소 번호가 뭐지?" 대비 둘 다 CER 0%)

### #3 (7F_KSK idx=4): "알려 줘" vs "알려줘"

| | 결과 |
|---|---|
| CSV (18k 테스트 당시) | 관리사무소 번호 알려 줘 |
| vpm_run (지금) | 관리사무소 번호 알려줘 |

- BPE 토크나이저 디코딩 경계 차이 (띄어쓰기만 다름)
- **영향:** CER 차이 없음 (공백 제거 후 동일)
